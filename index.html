《小狗钱钱》--读书笔记

发表于 2020-08-28 更新于 2020-09-05

最近虽然也读了不少书，可是丢掉了写读书笔记的习惯。但是，这也许和《how to read a book》有关系吧，因为是想要完整地读完这本书，写更加有结构和条理的读书笔记。

应该就是从现在开始吧，读书笔记的形式被改变了。从前只是希望这只是帮我整理书的结构并且自己能看懂就好了。可是现在，我要把它写成通识读物类，希望每个看我读书笔记的人都可以看懂这些书。这并不是什么很多视频做的所谓”干货”、“记好笔记“和”懒人包“，我个人十分排斥这种做法。我希望通过我的读书笔记，让你对于这本书有一个大概的了解，从而激发你自己花时间读这本书，这样更会打磨你的思维，更加容易滴理解和记住这本书。这也是我践行 费曼学习法 的一种方式吧。

小狗钱钱

这段是我读到52页之后的感想吧，没什么结构，只是想到哪里就说到哪里了。
在这个时间点之前，很早的以前，对于“钱”这个词，理解只是觉得以后有钱了会怎么怎么样，有钱了，最主要的是可以买我喜欢的品牌、单品之类的，有钱了，和女孩约会会很简单，有钱了，…… 最经典的一句话是，“我要是有钱，我会是这个逼样嘛？“。确实，不懂金钱概念的人，有钱了还是那个逼样。到了最近一年多，我知道了这些东西是要学习的，也是花了好多时间，犯了特别多的错误，才切身知道了，从书本上学习是最佳的方式。 只是不知道看一些什么样的书籍，来掌握这些技巧。到了现在，十分后悔过年时，得到的那3k，虽然数目很少，但懊悔有些钱的时候，对金钱没有概念，不会支配自己掌握的钱。但是，希望读完这本书能让我开始对于金钱有一个自己的掌控。

因为这本书讲的是一个又一个发生在女孩吉娅身上的故事，而且这本书页数也很少，因此内容会比较少。

去了解金钱，就像去了解女孩一样

很多人对于金钱几乎没什么概念，就像之前的我一样，钱是用来花的，钱可以买吃的，穿的，住的；可以在银行贷款；“我要是有钱了，我肯定……“。或许这是大多数人再为了解金钱之前的一些想法，就像很多屌丝说的，“她肯定是因为钱才和他在一起的，我要是有钱，他肯定会和我谈恋爱”。其实这个谈恋爱一样，你能不能和女孩谈恋爱，起关键作用的是你会不会和女孩聊天，调情，约会，而不是你有没有钱，这是两个事情。即使你很有钱，不会交往，还是很难留住人心。同样的，某些情况下，即使是拥有了钱，很多人也很难留住他们的钱。

梦想储蓄罐和梦想相册

我们赚钱的目的是为了用钱置换一些东西，而这些可能就是你的“梦想”，因此有一个梦想储蓄罐和梦想相册是很重要的，不是真的要你搞一个储蓄罐，而是要有一个单独的账户用来储蓄用于“梦想”的花费。而关于梦想相册，就是把想要的物品或者想去的地方的图片，制作成相册，然后想象你得到它，或者去那里玩的场景。关于这部分，我不止听过一个人讲“想象”的力量，可是，我个人对此还是有些怀疑的。

有了这些梦想，就尽快去行动，要在72小时之内行动，否则你就可能再也做不到了。【如果你只是带着试试看的心态，那么最后只会以失败而告终，你会一事无成。尝试纯粹是一种借口，你还没做，就已经给自己退路。不能试验，你只有两个选择，做，或者不做。】

读书笔记--《你的灯亮着吗？-发现问题的真正所在》

发表于 2020-05-20 更新于 2020-05-23

前言

这是我的第二个读书笔记，自从读完《刻意练习》以后，正在尽着最大的努力把刻意练习的规则融入生活和思维中。而这本书其实也是能解决问题的一本书，同样希望从这本书中学到方法论并且运用自如。都一个目的啦。

btw，这本书的序，在我认为完美的表达了这本书的中心思想。

问题是什么？

一个问题

首先通过一个事件来引出作者所要讲述的关键。

在哥谭市中心有一座73层的，刚刚建成的大楼。虽然这栋楼里还有很多空房，但是楼里的电梯已经不够用了，因此租户威胁说，如果不改善电梯，就退租。大厦里主要都是办公场所，每层的入住率都差不多，为了把剩余的办公室租出去，大厦投入了大量的广告费用，并且金融区狭小人口密度大，这一消息传播得非常快。

通常的思维，是解决问题的思维，直接提出解决问题的方案。但是先问自己几个问题再给出答案或许有更好的答案呢。

这是什么类型的问题？谁碰到了问题？问题是什么？或者说，此刻问题的本质是什么？

先来看看“谁碰到了问题“，这样问是为了：
（1） 确定服务对象，也就是弄清楚解决问题是为了让谁满意。
（2） 为了找到合适的解决方案提供一些线索。
首先是电梯的使用者碰到了问题，是电梯不够用的问题。其次，是大厦拥有者遇到的问题，大厦里的人对大厦不满意。

★问题是什么？

很多时候，问题的解决者还没【定义】好问题的时候就匆忙地给出了解决方案，尽管他们可以找到很多方法来解决问题，但是不一定对症。有的人是想定义好问题，可是在这件事上犹犹豫豫，不知道到底该把这个问题定义成什么类型。
在实际操作中，一方面，对日常生活中自然产生的问题，不可能给出独一无二、完全明确的定义；另一方面，如果不能就问题本身达成一定程度的共识，几乎可以肯定，最后找到的解决方案无法切中要害。这个时候，一般都是在为说话嗓门大、口才好或者资产最雄厚的人解决问题。

因此，入门的关键成了【把单一思维模式切换成多重思维模式】。
为了实现这一切换，问题解决者应该尽早回答以下两个问题：
1 谁碰到了问题？
2 问题的本质是什么？

题目：
给定一个含有 M x N 个元素的矩阵（M 行，N 列），请以对角线遍历的顺序返回这个矩阵中的所有元素，对角线遍历

示例：

★解题思路
第一次遇到二维数组的遍历，感觉和一维数组还是有些区别。没什么解题思路了，都写在了注释里，还是多做题。

★代码实现

public static int[] find(int[][] matrix){
       //检查是否为空数组
       if(matrix.length == 0 && matrix ==null){
           return new int[0];
       }

       int n = matrix.length; // 行
       int m = matrix[0].length; //列

       int row = 0, column = 0;

       int direction = 1; //对角线方向

       int[] res = new int[n * m]; // 接受的容器
       int r = 0;

       //对二维数组进行遍历
       while(row < n && column < m){

           //将遍历元素放入接收容器
           res[r++] = matrix[row][column];

           //对角线遍历，方向向上或向下
           int new_row = row + (direction == 1 ? -1 : 1);
           int new_column = column + (direction == 1 ? 1 : -1);

           //当对角线走到边界时，
           if(new_row < 0 || new_row == n || new_column < 0 || new_column == m){
               if(direction == 1){
                   //对角线是正方向时遇到的两种情况， 向右或向下
                   row += (column == m - 1 ? 1 : 0) ;
                   column += (column < m - 1 ? 1 : 0);

               }  else{
                   //对角线是负方向时遇到的两种情况， 向下或向右
                   column += (row == n -1 ? 1 : 0);
                   row += (row < n - 1 ? 1 : 0);

               }
               //因为已经到了边界，因此需要反转方向
               direction = 1 - direction;

           } else{

               //没有在边界时，按照对角线方向遍历即可
               row = new_row;
               column = new_column;

           }


       }

       return res;
   }
