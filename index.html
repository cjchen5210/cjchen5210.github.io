前言

这是我的第二个读书笔记，自从读完《刻意练习》以后，正在尽着最大的努力把刻意练习的规则融入生活和思维中。而这本书其实也是能解决问题的一本书，同样希望从这本书中学到方法论并且运用自如。都一个目的啦。

问题是什么？

一个问题

首先通过一个事件来引出作者所要讲述的关键。

在哥谭市中心有一座73层的，刚刚建成的大楼。虽然这栋楼里还有很多空房，但是楼里的电梯已经不够用了，因此租户威胁说，如果不改善电梯，就退租。大厦里主要都是办公场所，每层的入住率都差不多，为了把剩余的办公室租出去，大厦投入了大量的广告费用，并且金融区狭小人口密度大，这一消息传播得非常快。

通常的思维，是解决问题的思维，直接提出解决问题的方案。但是先问自己几个问题再给出答案或许有更好的答案呢。

这是什么类型的问题？谁碰到了问题？问题是什么？或者说，此刻问题的本质是什么？

先来看看“谁碰到了问题“，这样问是为了：
（1） 确定服务对象，也就是弄清楚解决问题是为了让谁满意。
（2） 为了找到合适的解决方案提供一些线索。
首先是电梯的使用者碰到了问题，是电梯不够用的问题。其次，是大厦拥有者遇到的问题，大厦里的人对大厦不满意。

★问题是什么？

很多时候，问题的解决者还没【定义】好问题的时候就匆忙地给出了解决方案，尽管他们可以找到很多方法来解决问题，但是不一定对症。有的人是想定义好问题，可是在这件事上犹犹豫豫，不知道到底该把这个问题定义成什么类型。
在实际操作中，一方面，对日常生活中自然产生的问题，不可能给出独一无二、完全明确的定义；另一方面，如果不能就问题本身达成一定程度的共识，几乎可以肯定，最后找到的解决方案无法切中要害。这个时候，一般都是在为说话嗓门大、口才好或者资产最雄厚的人解决问题。

因此，入门的关键成了【把单一思维模式切换成多重思维模式】。
为了实现这一切换，问题解决者应该尽早回答以下两个问题：
1 谁碰到了问题？
2 问题的本质是什么？

对角线遍历

题目：
给定一个含有 M x N 个元素的矩阵（M 行，N 列），请以对角线遍历的顺序返回这个矩阵中的所有元素，对角线遍历

示例：

★解题思路
第一次遇到二维数组的遍历，感觉和一维数组还是有些区别。没什么解题思路了，都写在了注释里，还是多做题。

★代码实现

public static int[] find(int[][] matrix){
       //检查是否为空数组
       if(matrix.length == 0 && matrix ==null){
           return new int[0];
       }

       int n = matrix.length; // 行
       int m = matrix[0].length; //列

       int row = 0, column = 0;

       int direction = 1; //对角线方向

       int[] res = new int[n * m]; // 接受的容器
       int r = 0;

       //对二维数组进行遍历
       while(row < n && column < m){

           //将遍历元素放入接收容器
           res[r++] = matrix[row][column];

           //对角线遍历，方向向上或向下
           int new_row = row + (direction == 1 ? -1 : 1);
           int new_column = column + (direction == 1 ? 1 : -1);

           //当对角线走到边界时，
           if(new_row < 0 || new_row == n || new_column < 0 || new_column == m){
               if(direction == 1){
                   //对角线是正方向时遇到的两种情况， 向右或向下
                   row += (column == m - 1 ? 1 : 0) ;
                   column += (column < m - 1 ? 1 : 0);

               }  else{
                   //对角线是负方向时遇到的两种情况， 向下或向右
                   column += (row == n -1 ? 1 : 0);
                   row += (row < n - 1 ? 1 : 0);

               }
               //因为已经到了边界，因此需要反转方向
               direction = 1 - direction;

           } else{

               //没有在边界时，按照对角线方向遍历即可
               row = new_row;
               column = new_column;

           }


       }

       return res;
   }

需要注意的就是第二层循环的次数，遍历是长度-1，因为最后一个元素不需要和自己比较，比较的次数是长度-1-i，因为比较的时候第n个元素就不用再和之前的元素比较了。因此冒泡排序是从最后一个元素开始倒着变成所要的顺序的（不知道你们理不理解我的意思）。这也是冒泡排序名字的由来(或许吧）。

★插入排序

✧原理
插入排序的原理和整理扑克牌的顺序基本一致。从第二个元素开始，与第一个元素比较，如果第一个元素大，第一个元素后移到第二个位置，将插入元素插入到第一位。到第n+1个元素时，前n个元素已经排好序，第n+1个从第n个元素进行比较，如果前一个大于后一个，前一个元素就后移，直到前一个小于后一个，停止循环，插入元素。

✧实现

★堆排序

✧原理
堆排序是通过数据结构堆来对数组进行排序，结合了插入排序和归并排序的优点。首先要建立一个堆，维护最大堆的性质，然后建立一个堆，再对堆内进行操作。

✧实现

★快速排序

✧原理
快速排序是通过设置一个基准数，先从后半部分开始，如果扫描到的值大于，扫描左移，如果小于基准数，就将这个数字放在第一位；然后从前半部分开始，如果比基准值小，扫描右移，如果比基准值大，就放在最后一位。
之后进行递归。
✧实现
通过while循环控制从后到前和从前到后的扫描，然后前后扫描值一样时，进行下一次递归。

★单链表

链表概念：
链表是数据结构的一种，其中的各对象按线性顺序排列。数组的线性顺序是由数组下标决定的，然而与数组不同的是，链表的顺序是由各个对象里的指针决定的。链表为动态集合提供了一种简单灵活的表示方法，并且能支持很多操作。
链表分为单链表和双链表。双链表会在以后的blog中更新。
（我猜的（欢迎批评指正）） 与链表平行的是【数组】

对于链表的操作包括【增、删、改、查】

✧增

对于向链表中添加元素，有两种方式，包括在链表的最后增加节点，以及在链表中间插入节点。
通过一个又一个节点的插入，就形成了一个单链表。
而【单链表的节点】包括：存储的数据和指针（指针指向下一个节点的地址）。【头节点】是链表的第一个节点，它可以不存储元素，【首节点】是链表第一个存储数据的节点，头节点和首节点可以是一个，但是我个人会将它们分开使用。
